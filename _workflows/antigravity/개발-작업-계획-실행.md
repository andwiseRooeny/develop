## ⚠️ 핵심 원칙: 빠짐없는 기록

> [!CAUTION]
> **모든 대화, 모든 수정, 모든 요청은 반드시 기록되어야 합니다.**
> - 사용자와의 대화 흐름 전체를 `Status.md`에 누적 기록
> - 사소한 요청이라도 누락 금지
> - 어떤 코드 수정이든, 어떤 추가 요청이든, 모두 **실시간 반영**
> - 다음 워크플로우 호출은 **사용자가 명시적으로 지시**할 때만 수행

---

### 실행 단계

1. **[현황 및 구현 위치 파악]**
   - 사용자에게 [업무명]을 입력받아 `_develop/_ing/[날짜]_[업무명]_Status.md` 파일을 생성합니다.
   - **[Dashboard 등록]**: 생성된 `Status.md` 링크를 `_develop/_ing/_DASHBOARD.md`의 **In-Progress Tasks** 섹션에 등록합니다.
   - 작업을 시작하기 전, 반드시 `_develop/_info/Work_Directory.md`를 참조하여 현재 프로젝트의 디렉토리 구조와 각 폴더의 역할을 확인합니다.

2. **[상세 수정 계획 수립 (필수)]**
   - 소스 코드를 수정하기 전, 다음 내용을 포함한 계획을 수립하여 매크로/텍스트로 정리합니다.
     - **수정 대상 파일 및 경로**
     - **어떤 로직을 구체적으로 어떻게 수정할 것인지**
     - **예상되는 영향 범위와 사이드 이펙트**
   - 이 계획은 반드시 `Status.md` 또는 `implementation_plan.md`에 기록되어야 합니다.

3. **[사용자 계획 검토 및 실행 승인]**
   - 수립된 계획을 사용자에게 제시하고 **"실행 여부"**를 반드시 확인받습니다.
   - 사용자의 응답에 따라:
     - **추가 질의/수정**: 계획을 보완하고 다시 승인을 요청합니다.
     - **실행 승인**: 다음 단계인 '실행'으로 전환합니다.

4. **[작업 실행 및 실시간 기록]**
   - 승인된 계획에 기반하여 [헤더]의 **맥락 격리 및 내부 지식 우선** 원칙을 준수하며 작업을 진행합니다.
   
   > [!IMPORTANT]
   > **실시간 기록 의무**
   > - 코드 수정 직후 → `Status.md`의 **수정 파일 목록** 즉시 갱신
   > - 사용자 추가 요청 발생 시 → `Status.md`의 **대화 로그** 섹션에 즉시 기록
   > - 새로운 요구사항 반영 시 → **요구사항** 섹션 즉시 업데이트
   
   - **Records**:
     - 참조 문서(`_ref`, `_lib`, `_info`)는 반드시 상대 경로 링크로 기록합니다.
     - 수정된 파일 목록은 프로젝트 루트 기준 상대 경로로 누적합니다.
   - **중요**: 중간 마일스톤이 아니더라도, **모든 변경 사항**을 `Status.md`에 즉시 반영합니다.

5. **[후속 요청 처리]**
   - 작업 도중 사용자가 추가 요청을 하면:
     1. 해당 요청을 `Status.md`의 **대화 로그** 또는 **추가 요청** 섹션에 기록
     2. 요청된 작업 수행
     3. 수행 결과를 `Status.md`에 즉시 반영 (수정 파일 목록, Walkthrough 등)
   - **절대 금지**: 기록 없이 다음 작업으로 넘어가는 것

6. **[작업 종료 및 Walkthrough 통합]**
   - **사용자가 작업 종료를 명시적으로 선언**할 때까지 작업은 진행 중 상태를 유지합니다.
   - 작업 완료 시, **Walkthrough 섹션**을 `Status.md` 하단에 직접 통합하여 기록합니다.
   - Walkthrough 필수 포함 요소:
     - **수정 내용 요약**: 변경된 코드의 diff 블록 또는 핵심 코드 스니펫
     - **동작 흐름**: Mermaid 다이어그램(sequenceDiagram 등)으로 시각화
     - **검증 방법**: 테스트 절차 및 예상 결과
     - **참조 문서**: 관련 레퍼런스 링크
   - **[Dashboard 업데이트]**: `_DASHBOARD.md`에서 해당 작업의 상태를 `[x] 완료`로 변경하거나 섹션을 이동합니다.
   - **다음 워크플로우 호출 금지**: `/개발-결과물-저장` 등 후속 워크플로우는 **사용자가 직접 호출**해야만 실행합니다.
