# 레퍼런스 문서 템플릿 (Reference Document Template)

본 문서는 `_ref/` 하위에 생성되는 모든 레퍼런스 문서의 **표준 템플릿**을 정의합니다.

---

## 📄 표준 템플릿 구조

모든 레퍼런스 문서는 다음 템플릿 구조를 따릅니다:

```markdown
# [레퍼런스 제목]

## 📐 분류 정보 (Classification)

| 항목 | 값 |
|:---|:---|
| **계층 (Layer)** | [Presentation/Application/Business/Persistence/Infrastructure] |
| **분류 코드** | [pres-*/app-*/biz-*/pers-*/infra-*] |
| **저장 유형** | [Common/Custom] |

## 종합 요약 및 노하우
[전체 대상의 역할과 기술 정수를 선언적으로 요약 - 1~2줄]

## Referenced Assets
[분석된 모든 파일/디렉토리의 상대 경로 리스트]

## 핵심 선언
### [선언 항목 1]
[핵심 내용 - 1~2줄]

### [선언 항목 2]
[핵심 내용 - 1~2줄]

## Source Specifics (필요 시)
[핵심 로직 시그니처 코드 블록]
```

---

## ✅ 필수 섹션

| 섹션 | 필수 여부 | 설명 |
|:---|:---|:---|
| `# 제목` | ✅ 필수 | 레퍼런스 주제를 명확히 표현 |
| `## 📐 분류 정보` | ✅ 필수 | 계층, 분류 코드, 저장 유형 테이블 |
| `## 종합 요약 및 노하우` | ✅ 필수 | 1~2줄 선언적 요약 |
| `## Referenced Assets` | ✅ 필수 | 분석된 소스 경로 리스트 |
| `## 핵심 선언` | ✅ 필수 | 선택된 옵션별 상세 선언 |
| `## Source Specifics` | 선택 | 핵심 코드 블록 (필요 시) |

---

## 📝 작성 규칙

1. **장황한 설명 절대 금지** - 항목당 1~2줄 이내
2. **코드 블록은 핵심 시그니처만** - 전체 코드 복사 금지
3. **`## 📐 분류 정보` 섹션은 문서 상단**에 위치
4. **상대 경로 사용** - 프로젝트 루트 기준

---

## 📊 인덱스 테이블 포맷

`_Ref_Index.md` 파일 작성 시 다음 테이블 형식을 사용합니다:

```markdown
| 분류 | 계층 | 분류 코드 | 주제 | 내용 요약 | 핵심 기술 | 링크 |
|:---|:---|:---|:---|:---|:---|:---|
| Common | Application | app-controller | 로그인 처리 | 사용자 인증 및 세션 관리 | Spring MVC | [파일명](./common/파일명.md) |
```

### 인덱스 정렬 규칙
1. **분류**: Common 우선, Custom 후순
2. **계층 순서**: Presentation → Application → Business → Persistence → Infrastructure
3. **주제명**: 가나다/ABC 순
