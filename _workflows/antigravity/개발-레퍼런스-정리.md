### 실행 단계

0. **[헤더 참조 및 세계관 동기화] (CRITICAL)**
   - 워크플로우 상단에 정의된 모든 **Header(참조 문서)**를 식별하고, 해당 파일들을 반드시 `view_file` 도구로 열어 내용을 확인합니다.
   - 각 문서에 정의된 규칙, 명명법, 분류 체계 등의 **세계관(Worldview)**을 완벽히 숙지하고 이를 절대적으로 준수하여 업무를 수행합니다.


0. **[프로젝트 루트 확인]**
   - 실행 전 반드시 `_develop/_info/Project_Root.md` 파일이 존재하는지 확인합니다.
   - 프로젝트 루트를 기준으로 모든 경로를 상대 경로로 계산합니다.

1. **[스캔 및 분류]**
   - 스캔 대상: `_develop/_ref/common/` 및 `_develop/_ref/custom/`
   - 각 폴더 내의 `.md` 파일을 수집하고 명칭과 경로를 확보합니다.

2. **[내용 분석 및 데이터 추출]**
   - 각 `.md` 파일을 읽어 다음 정보를 추출합니다.
     - **주제**: `#` (1단계 헤더) 제목
     - **계층/분류 코드**: `## 📐 분류 정보` 섹션에서 추출 (없을 경우 파일명 또는 내용 기반 추론)
     - **내용 요약**: `# 종합 요약 및 노하우` 섹션의 내용 (또는 본문 상단 1~2줄)
     - **핵심 기술**: 본문 내의 핵심 키워드(예: Spring, MyBatis, SSO, API 등) 또는 코드 블록의 언어 기반 추출

3. **[인덱스 파일 생성/갱신]**
   - 대상 파일: `_develop/_ref/_Ref_Index.md`
   - **[Architecture]**의 `📊 인덱스 테이블 포맷` 섹션을 준수하여 작성합니다.
   - 링크는 반드시 **상대 경로(`./common/파일명.md`)**를 사용합니다.

4. **[결과 보고]**
   - 생성된 `_Ref_Index.md` 파일을 사용자에게 보여주고 작업을 종료합니다.
