이 워크플로우는 `_develop/_lib` 하위의 라이브러리 문서들을 분석하여, 핵심 기술과 솔루션이 포함된 목차 인덱스(`_Lib_Index.md`)를 생성하거나 관리합니다.

**[Header]**: [개발-공통-준수-규칙](./개발-공통-준수-규칙.md)

---

### 실행 단계

0. **[헤더 참조 및 세계관 동기화] (CRITICAL)**
   - 워크플로우 상단에 정의된 모든 **Header(참조 문서)**를 식별하고, 해당 파일들을 반드시 `view_file` 도구로 열어 내용을 확인합니다.
   - 각 문서에 정의된 규칙, 명명법, 분류 체계 등의 **세계관(Worldview)**을 완벽히 숙지하고 이를 절대적으로 준수하여 업무를 수행합니다.


0. **[프로젝트 루트 확인]**
   - 실행 전 반드시 `_develop/_info/Project_Root.md` 파일이 존재하는지 확인합니다.
   - 프로젝트 루트를 기준으로 모든 리소스 경로를 상대 경로로 계산합니다.

1. **[파라미터 및 모드 결정]**
   - 사용자가 입력한 추가 파라미터(정리 주제)가 있는지 확인합니다.
   - **Case A (Default)**: 파라미터가 없다면 → **Step 3 (전체 인덱싱)**로 이동.
   - **Case B**: 파라미터(키워드)가 있다면 → **Step 4 (주제별 큐레이션)**로 이동.

2. **[라이브러리 스캔]**
   - `list_dir`을 사용하여 `_develop/_lib` 내의 모든 `.md` 파일을 수집합니다. (인덱스 파일 제외)

3. **[Case A: 전체 인덱싱 모드 고도화]**
   - 각 라이브러리 파일을 읽어 다음 정보를 추출합니다.
     - **작업 주제**: 파일 상단 제목 (#)
     - **핵심 솔루션 요약**: `# 종합 요약 및 노하우` 또는 본문 상단 1~2줄
     - **핵심 기술**: 본문 내 주요 기술 키워드 (Java, MyBatis, SSO, API 등)
   - `_develop/_lib/_Lib_Index.md` 파일을 테이블 형식으로 갱신합니다.
     - `| 작업 주제 | 핵심 솔루션 요약 | 핵심 기술 | 링크 |`
     - 링크는 반드시 **상대 경로(`./파일명.md`)**를 사용합니다.

4. **[Case B: 주제별 큐레이션 모드]**
   - 입력받은 키워드와 연관된 라이브러리들을 선별 및 분석합니다.
   - 선별된 파일들을 기반으로 공통된 **문제 해결 패턴**을 추출하여 전용 큐레이션 문서(`_Curation_[주제].md`)를 생성합니다.

5. **[결과 보고]**
   - 생성/갱신된 인덱스 파일을 사용자에게 보여주며 작업을 마무리합니다.
